## 2. 技术说明

- **前端**：原生 HTML5 + CSS3 + JavaScript (Canvas 2D)
- **初始化**：单 HTML 文件项目，使用 `<canvas>` 渲染游戏画面
- **字体**：Google Fonts - "Press Start 2P"（像素字体）
- **后端**：无（纯前端游戏）
- **构建工具**：无（直接运行 HTML 文件，或通过 VSCode Live Server 启动）

## 4. 核心类设计

### 4.1 Mecha 类
```
class Mecha {
  - x, y: 位置坐标
  - width, height: 尺寸
  - hp: 血量 (max 100)
  - direction: 朝向 (-1 左, 1 右)
  - state: 状态 (idle, walking, attacking, defending, hit, jumping)
  - playerNumber: 玩家编号 (1 或 2)
  - color: 颜色方案 (红/蓝)
  - attackCooldown: 攻击冷却计时器
  - hitStunTimer: 受击硬直计时器
  
  + update(deltaTime): 更新状态
  + render(ctx): 绘制机甲
  + move(direction): 移动
  + attack(): 攻击动作
  + defend(): 防御动作
  + jump(): 跳跃动作
  + takeDamage(amount): 受击扣血
  + isAlive(): 存活判定
}
```

### 4.2 CombatSystem 类
```
class CombatSystem {
  - mecha1, mecha2: 两个机甲引用
  
  + update(): 更新战斗逻辑
  + checkHit(attacker, defender): 检测攻击命中
  + calculateDamage(attacker, defender): 计算伤害
  + applyKnockback(attacker, defender): 应用击退效果
  + checkWinCondition(): 检查胜负条件
}
```

### 4.3 InputHandler 类
```
class InputHandler {
  - keys: 当前按键状态 Map
  
  + constructor(): 注册键盘事件
  + isKeyDown(key): 检测按键
  + getPlayer1Input(): 获取玩家1输入
  + getPlayer2Input(): 获取玩家2输入
  + destroy(): 移除事件监听
}
```

### 4.4 SceneManager 类
```
class SceneManager {
  - currentScene: 当前场景
  - scenes: 场景注册表
  
  + switchTo(sceneName): 切换场景
  + update(deltaTime): 更新当前场景
  + render(ctx): 渲染当前场景
}
```

## 5. 游戏循环

```
主循环 (60fps requestAnimationFrame):
  1. 处理输入 (InputHandler)
  2. 更新游戏状态 (Mecha.update, CombatSystem.update)
  3. 碰撞检测 (CombatSystem.checkHit)
  4. 伤害计算与状态变更
  5. 胜负判定 (CombatSystem.checkWinCondition)
  6. 渲染场景 (Scene.render)
  7. 渲染 HUD (血量条、信息)
  8. 渲染机甲动画 (Mecha.render)
  9. requestAnimationFrame(主循环)
```

## 6. 数据模型

### 6.1 游戏状态
```
GameState {
  phase: 'menu' | 'battle' | 'result'  // 游戏阶段
  winner: null | 1 | 2                  // 获胜者
  frameCount: number                    // 帧计数器
}
```

### 6.2 机甲状态
```
MechaState {
  hp: number                            // 当前血量 (0-100)
  maxHp: number = 100                   // 最大血量
  x: number                             // X 坐标
  y: number                             // Y 坐标
  vx: number                            // X 方向速度
  vy: number                            // Y 方向速度
  width: number                         // 宽度 (像素块)
  height: number                        // 高度 (像素块)
  facing: -1 | 1                        // 朝向
  state: 'idle' | 'walk' | 'attack' | 'defend' | 'hit' | 'jump'
  animFrame: number                     // 动画帧索引
  animTimer: number                     // 动画计时器
  attackTimer: number                   // 攻击持续计时
  cooldownTimer: number                 // 冷却计时
  hitStunTimer: number                  // 硬直计时
}
```

### 6.3 像素角色定义
机甲使用 16x16 像素网格绘制，通过 2D 数组定义形状：
- 身体：矩形块（主色）
- 头部：小矩形（较亮色）
- 腿部：两个小矩形（主色深色）
- 手臂：身体两侧的细条（辅色）

## 7. 基础素材方案

### 7.1 像素机甲绘制方案
所有机甲角色采用 Canvas 程序化绘制（像素块拼凑），无需外部图片资源：
- **身体**：6x8 像素块，主色填充，边框加深
- **头部**：4x4 像素块，亮色 + 眼部（白色小点）
- **手臂**：2x6 像素块，辅色，可前后摆动
- **腿部**：3x4 像素块*2，深色，迈步动画
- **武器**：攻击时伸出 4x2 像素条

### 7.2 场景绘制方案
- **地面**：Canvas 绘制水平线和间隔装饰点
- **背景**：渐变深色 + 随机闪烁白色小点（星空效果）
- **边界**：左右两侧垂直装饰线

### 7.3 特效方案
- **攻击特效**：白色/黄色闪光像素条
- **防御特效**：半透明六边形/圆形护盾光晕
- **受击特效**：红色闪烁 + 碎片飞溅小点
